======================================================================
ノベルゲームエンジン プロジェクト構成および設計ルール
======================================================================

【 1. プロジェクト概要 】
HTML5 CanvasとDOM要素を組み合わせた、Webベースのノベルゲームエンジンです。

最大の特徴は「Googleスプレッドシートをマスターデータとして構築する」点にあります。
ゲームの進行（シナリオ）や、UIのデザイン設定（CONFIG）はすべてスプレッドシート上で管理され、
GAS（Google Apps Script）経由でJSONとして読み込み、ゲームを動的に生成します。

シナリオの途中で「ミニゲーム（クイズなど）」を iframe 経由で呼び出し、
その結果をノベルゲーム側に還元する機能を備えています。


----------------------------------------------------------------------
【 2. スプレッドシートのデータ構造 】

本エンジンは、以下の2つのシートから出力されたデータ（配列オブジェクト）で駆動します。

■ シナリオ管理シート (SCENARIO)
ゲームの進行タイムラインを定義します。
主な列：cmd, name, text, text_rubi, cond, font, fontsize, color, outline, pos, dir, src, to, voice_type, voice_id, duration
（※ cmd列が bgm, bg, show, effect, choice, minigame などの命令として解釈されます）

■ CONFIG管理シート (CONFIG)
ゲーム全体の設定やUIのスタイルを定義します。
主な列：name, text, color, font, fontsize, outline, pos, dir, src
（※ name列が main_color, font_main, title_bg, se_click などの設定キーとして解釈されます）

【 3. ディレクトリ・ファイル構成 】

/project-root
 ├── index.html               (ベースとなるUI構造・外部ファイル読み込み)
 ├── style.css                (UIデザイン・アニメーション定義)
 ├── app.js                   (統合されたメインロジック・各種クラス群)
 ├── scenario.js              (任意：ローカルテスト用のシナリオデータ)
 ├── minigame_quiz.html       (ミニゲームの例：クイズ。これらは1ファイル完結)
 │
 ├── images/                  (背景、キャラクター、UIパーツなどの画像素材)
 └── sounds/                  (BGM、SE、ボイスなどの音声素材)


----------------------------------------------------------------------
【 4. データの流れ（初期化フロー） 】

  1. index.html が style.css と app.js などをロードする。
  2. app.js 内の初期化処理が起動し、DataLoader を用いて GAS または 
     ローカルの scenario.js からJSONデータを取得する。
  3. 取得したデータを SCENARIO および CONFIG 変数に格納する。
  4. DataLoader が CONFIGシート の値（color, font, posなど）を読み取り、
     DOMのインラインスタイルやCSS変数へ動的に適用する。
  5. アプリケーションが素材をキャッシュし、タイトル画面を表示してユーザー入力を待機する。

【 5. ミニゲームの設計・実装ルール 】

シナリオシートの「cmd: minigame」によって iframe 形式で呼び出されるミニゲーム
（例：minigame_quiz.html）を作成・修正する際は、以下のルールを厳守してください。

■ Single-File Portability（1ファイル完結）
  ミニゲームは本体エンジンとは異なり、HTML/CSS/JS を【1ファイル内】にすべて
  記述してください。外部のCSS/JSファイルは読み込まないでください。

■ Configuration Driven（設定の分離）
  マジックナンバーを避け、ゲームの難易度やスコア設定などは 
  USER_SETTINGS オブジェクトとしてJSの先頭で定義してください。

■ Class-Based OOP & Safety
  ロジックはクラスにカプセル化し、DOMContentLoaded 内で安全に初期化してください。
  ゲーム終了時は必ず destroy() メソッド等でイベントリスナーを解除してください。

■ postMessage による結果の返却
  ミニゲーム終了時、スプレッドシート側のフラグ処理と連携するため、
  必ず以下の形式で親ウィンドウ（ノベルゲーム本体）に結果を送信してください。

  window.parent.postMessage({
    type: 'MINIGAME_END',
    flags: { minigame_score: this.score, minigame_clear: this.isClear }
  }, '*');
